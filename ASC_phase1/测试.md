# ASC 三个实验层次测试命令清单

这份文件按三种实验层次整理命令：

1. 原先三个论文模型：使用作者提供的 `.pt` 向量、默认 layer、默认 gamma。
2. 原先三个论文模型：重新生成 pairs、提取新向量、使用默认 layer、重新提取新 gamma。
3. 全新模型：模型、pairs、向量、layer、gamma、输出路径全部显式指定。

最重要的规则：

```text
提取引导向量、提取 gamma、最终评测必须使用同一个 layer_index。
```

prompt 口径也要固定：

```text
DeepSeek-R1-Distill-Qwen-7B / DeepSeek-R1-Distill-Llama-8B / QwQ-32B 使用 paper_cot。
Qwen3-8B 使用 chat_boxed_cot，并开启 qwen3_enable_thinking。
```

如果换到全新模型，先看模型官网或模型卡。不同模型是否需要 chat template、thinking mode、`\boxed{}` 格式，不能靠旧模型命令猜。

默认配置只服务原先三个论文模型：

| 模型 | 默认 layer | 作者向量 | 默认 gamma |
|---|---:|---|---:|
| DeepSeek-R1-Distill-Qwen-7B | 20 | `vectors/steering_vectors_qwen7b.pt` | 0.27 |
| DeepSeek-R1-Distill-Llama-8B | 20 | `vectors/steering_vectors_llama8b.pt` | 0.46 |
| QwQ-32B | 57 | `vectors/steering_vectors_qwq32b.pt` | 0.50 |

全新模型不要写进默认配置里自动猜。

## 第一层次：原模型 + 作者向量

适用场景：

```text
只想快速复现/检查原论文三个模型在某个数据集上的 CoT 和 ASC 效果。
```

这一层次不需要传：

```text
--steering_vector_path
--layer_index
--gamma_override
```

因为脚本会从原论文三模型的默认配置里读取作者向量、默认层号和默认 gamma。

### Qwen-7B / GSM8K

```bash
python eval_asc_paper.py \
  --model_name /root/autodl-tmp/deepseek-ai/DeepSeek-R1-Distill-Qwen-7B \
  --dataset gsm8k \
  --limit 200 \
  --prompt_mode paper_cot \
  --candidate_gammas 0,0.27 \
  --batch_size 24 \
  --attn_impl flash_attention_2 \
  --per_gamma_output_dir results/original_vector_qwen7b_gsm8k_200
```

### Llama-8B / GSM8K

```bash
python eval_asc_paper.py \
  --model_name /root/autodl-tmp/deepseek-ai/DeepSeek-R1-Distill-Llama-8B \
  --dataset gsm8k \
  --limit 200 \
  --prompt_mode paper_cot \
  --candidate_gammas 0,0.46 \
  --batch_size 16 \
  --attn_impl flash_attention_2 \
  --per_gamma_output_dir results/original_vector_llama8b_gsm8k_200
```

### QwQ-32B / GSM8K

```bash
python eval_asc_paper.py \
  --model_name /root/autodl-tmp/Qwen/QwQ-32B \
  --dataset gsm8k \
  --limit 200 \
  --prompt_mode paper_cot \
  --candidate_gammas 0,0.5 \
  --batch_size 4 \
  --attn_impl flash_attention_2 \
  --per_gamma_output_dir results/original_vector_qwq32b_gsm8k_200
```

说明：

- `0` 表示普通 CoT，不启用 ASC hook。
- `0.27/0.46/0.5` 是作者原配置里的默认 ASC gamma。
- 每个 gamma 跑完都会单独保存 JSON。
- 如果测 MATH，把 `--dataset gsm8k` 改成 `--dataset math`，同时适当降低 `--batch_size`。
- `--device_map` 和 `--max_memory` 默认不需要传，脚本会根据可见 GPU 自动处理；显存不够时再看文末“设备与显存参数”。

## 第二层次：原模型 + 新向量

适用场景：

```text
模型还是原论文三个模型之一，但你重新生成了长短 CoT pairs，并想提取自己的新 steering vector。
```

这一层次的原则：

- layer 仍使用该模型默认层。
- 但命令里建议显式写 `--layer_index`，避免之后看结果时不知道用了哪一层。
- 新向量、新 gamma、输出路径都必须显式写。

原三模型的新向量层号建议：

| 模型 | `--model_name` | 新向量 layer |
|---|---|---:|
| Qwen-7B | `/root/autodl-tmp/deepseek-ai/DeepSeek-R1-Distill-Qwen-7B` | 20 |
| Llama-8B | `/root/autodl-tmp/deepseek-ai/DeepSeek-R1-Distill-Llama-8B` | 20 |
| QwQ-32B | `/root/autodl-tmp/Qwen/QwQ-32B` | 57 |

下面以 Qwen-7B 为例。换 Llama/QwQ 时，替换 `--model_name`、文件名里的模型标签、`--layer_index` 和 batch size。

  --top_p 0.9 \

- `short_cot` 是否给出答案。

## 设备、注意力和长度参数说明

### `--device_map` 和 `--max_memory`

常规命令里不要传这两个参数。

它们的含义是：

- `--device_map`：告诉 Transformers 怎么把模型切到 GPU 上。`auto` 会自动判断可见 GPU；`balanced` 会尽量把模型分摊到多张 GPU。
- `--max_memory`：给每张 GPU 设置显存上限，例如 `0:20GiB,1:20GiB` 表示最多让模型在第 0/1 张卡各用 20GiB。

它们主要是手动显存调优参数，不是实验语义参数。默认情况下：

- 单卡：脚本/Transformers 会把模型放到单卡。
- 双卡：评测脚本会自动选择多卡布局；其他脚本使用 Transformers 的 `auto`。
- 只有 OOM 或显存分配明显不合理时，才手动加 `--device_map balanced --max_memory ...`。

如果确实 OOM，可以临时加：

```bash
--device_map balanced \
--max_memory 0:20GiB,1:20GiB
```

`0:16GiB,1:16GiB` 和 `0:20GiB,1:20GiB` 只是显存上限不同。之前 KL 诊断写 `16GiB` 是为了给有限差分/JVP 的中间张量留余量；生成和评测写 `20GiB` 是因为普通生成显存更稳定。现在命令清单不再默认写死它们。

### `--attn_impl`

建议口径：

- 生成 pairs、提取向量、最终评测：优先 `flash_attention_2`，和实际生成速度/显存更匹配。
- `extract_optimal_gamma.py`：只是前向统计 hidden norm，也可以用 `flash_attention_2`，所以命令里统一用它。
- `extract_paper_kl_gamma.py`：涉及有限差分、JVP/HVP 或曲率估计，`flash_attention_2` 对二阶/数值诊断不一定稳定，所以命令里用 `sdpa`。如果要严格验证二阶导，才考虑 `eager`，但更容易 OOM。

### `--max_input_tokens`

这是校准/提向量时输入上下文的最大长度，不是生成长度。

例如：

```text
long_prompt + long_cot
```

会被 tokenizer 截断到 `--max_input_tokens` 以内。

当前命令统一使用 `8192`，目的是尽量保留完整 long/short CoT。提取向量、hidden-norm gamma 和 KL 诊断都应该保持同一个上下文长度口径。

## 最小检查清单

### 第一层次检查

- 模型是否属于原论文三个模型。
- 是否真的想用作者原向量。
- `--candidate_gammas` 是否包含 `0` 和默认 gamma。
- 输出目录是否标明 `original_vector`。

### 第二层次检查

- pairs 是否是新生成的。
- checked pairs 是否已经人工剔除无效样本。
- 新向量 metadata 里的 `layer_index` 是否和评测一致。
- 评测是否显式传了新 `--steering_vector_path`。
- 输出目录是否标明 `new_vector` 和 layer。

### 第三层次检查

- 模型路径是否正确：`--model_name /path/to/model`。
- tokenizer 是否来自同一个模型路径。
- `generate_cot_pairs.py` 生成的 `long_cot` 是否完整。
- 新模型的 `--layer_index` 是否明确。
